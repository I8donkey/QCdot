# QBlock - 使用指南

## 版本

当前版本：1.0.0

## 基本概念

### 节点

节点是 QBlock 中的基本构建块。每个节点代表一个函数、变量或 UI 组件。节点具有：
- **输入端口**：节点的左侧（接收数据）
- **输出端口**：节点的右侧（生成数据）
- **标题**：节点类型的名称

### 连接

连接将输出端口链接到输入端口，创建数据流。连接按数据类型进行颜色编码。

### 数据类型

- **整数**：整数（1、2、3、...）
- **浮点数**：小数（1.5、3.14、...）
- **布尔值**：真/假值
- **字符串**：文本值
- **通用**：任何数据类型

## 节点面板

左侧面板显示所有可用节点，按类别组织：

1. **输入**：常量和变量节点
2. **输出**：打印和显示节点
3. **容器**：窗口和布局节点
4. **控件**：UI 组件节点
5. **控件方法**：控件操作节点
6. **数学**：算术运算
7. **逻辑**：比较和布尔运算
8. **字符串**：字符串操作
9. **转换**：类型转换
10. **文件**：文件操作
11. **工具**：工具（延时、随机数等）

### 创建节点

1. **拖放**：将节点从面板拖到画布上
2. **内联选择器**：将连接线拖到空白处并选择节点类型
3. **双击**：在空白画布上双击打开节点选择器

### 连接节点

1. 单击输出端口（右侧）
2. 拖动到输入端口（左侧）
3. 释放以创建连接

### 删除节点

1. 选择要删除的节点
2. 按 `Delete` 键或点击删除按钮

### 编辑节点值

双击常量节点以编辑其值。

## 执行模式

### 数据流模式（默认）

当所有输入都有有效值时，节点执行。执行根据数据可用性自动流动。

### 信号模式

节点从左到右执行，沿着显式信号连接。适用于命令式逻辑。

## 文件操作

### 保存

1. `Ctrl+S` 或 文件 → 保存
2. 选择位置和文件名
3. 图形保存为 `.qcd` 文件

### 打开

1. `Ctrl+O` 或 文件 → 打开
2. 选择 `.qcd` 文件
3. 图形加载到编辑器中

### 导出为 C++

1. 文件 → 导出为 C++
2. 选择生成源代码的位置

## 主题和语言

### 切换主题

使用工具栏按钮或菜单选项在深色和浅色主题之间切换。

### 切换语言

使用语言菜单在英语和中文之间切换。

## 高级功能

### 变量节点

使用变量节点在图形中存储和检索值：

1. 创建一个变量节点
2. 连接以读取或写入值
3. 变量在执行之间保持不变

### 条件执行

使用 IfElse 节点根据条件分支执行：

1. 添加一个 IfElse 节点
2. 将布尔条件连接到 "condition" 输入
3. 将节点连接到 "true" 和 "false" 分支

### 容器节点

使用容器节点创建结构化 UI 布局：

1. 添加主窗口或标签页控件容器
2. 添加布局节点
3. 添加控件并将其连接到布局
4. 使用方法节点操作容器

## 故障排除

### 无法创建连接

- 检查数据类型是否兼容
- 检查是否正在连接到输入（而不是输出到输出）

### 节点不执行

- 确保所有必需的输入都已连接
- 检查信号模式中是否有循环依赖

### 应用程序崩溃

- 经常保存您的工作
- 检查控制台输出中的错误消息

## API 参考

### 节点类型

所有节点类型都在 `nodes/include/QBlock/BuiltinNodes.h` 中定义。

### 引擎 API

核心执行引擎位于 `engine/include/QBlock/`。

### 编辑器 API

编辑器组件位于 `editor/include/QBlock/`。
